# Project Templates

In this tutorial, you will learn to create DOL projects using VUDO's template system. Templates provide pre-configured project structures optimized for different use cases.

## Overview

The `vudo new` command creates projects from templates:

```bash
vudo new <project-name> --template <template-name>
```

VUDO includes three built-in templates:
- **basic** - Minimal DOL project structure
- **ai-image** - AI image generation with spirit bindings
- **ai-text** - AI text/LLM projects with spirit bindings

## The Basic Template

The `basic` template is the default and provides a minimal DOL project structure.

<Terminal title="Creating a basic project">
```
$ vudo new my-app
Creating project 'my-app'...
  Using template: basic
  Creating directory structure...

my-app/
  project.dol          # Main project manifest
  src/
    domain.dol         # Domain entities
    lib.dol            # Library exports
  .dolrc               # Project configuration
  .gitignore           # Git ignore rules

Project created successfully!

Next steps:
  cd my-app
  vudo repl
```
</Terminal>

### Basic Template Structure

<FileTree>
```
my-app/
+-- project.dol         # Project manifest with version
+-- src/
|   +-- domain.dol      # Your domain entities
|   +-- lib.dol         # Public exports
+-- .dolrc              # VUDO configuration
+-- .gitignore          # Standard ignores
```
</FileTree>

### Generated Files

<CodeBlock title="project.dol" language="dol">
```dol
module my-app @ 0.1.0

// Project manifest - define your system here

requires dol/std @ ^1.0.0

system MyApp {
    uses domain.*

    docs {
        Main system for the my-app project.
        This is where you compose your domain genes into a complete system.
    }
}
```
</CodeBlock>

<CodeBlock title="src/domain.dol" language="dol">
```dol
module my-app/domain @ 0.1.0

// Domain entities go here

pub gen Entity {
    has id: UUID
    has created_at: Timestamp
    has updated_at: Timestamp

    is identifiable
    is auditable

    docs {
        Base entity gen with common fields.
        Extend this for your domain-specific entities.
    }
}
```
</CodeBlock>

<CodeBlock title=".dolrc" language="toml">
```toml
[project]

version = "0.1.0"
edition = "2024"

[check]
strict = true

[fmt]

```
</CodeBlock>

## The AI-Image Template

The `ai-image` template is designed for AI image generation projects. It includes spirit bindings for image AI providers.

<Terminal title="Creating an AI image project">
```
$ vudo new art-generator --template ai-image
Creating project 'art-generator'...
  Using template: ai-image
  Creating directory structure...
  Installing spirit bindings...

art-generator/
  project.dol
  src/
    domain.dol
    prompts.dol         # Prompt engineering genes
    generation.dol      # Image generation traits
  spirits/
    openai.spirit       # OpenAI DALL-E binding
    stability.spirit    # Stability AI binding
    replicate.spirit    # Replicate binding
  .env.example          # Environment template
  .dolrc
  .gitignore

Project created successfully!

Next steps:
  cd art-generator
  cp .env.example .env
  # Add your API keys to .env
  vudo repl
```
</Terminal>

<FileTree>

```
art-generator/
+-- project.dol
+-- src/
```

### Key Files

<CodeBlock title="src/prompts.dol" language="dol">
```dol
module art-generator/prompts @ 0.1.0

uses std/text

pub gen Prompt {
    has content: String
    has style: PromptStyle?
    has negative: String?

    docs {
        A structured prompt for image generation.
        Supports style modifiers and negative prompts.
    }
}

pub enum PromptStyle {
    Photorealistic,
    Artistic,
    Anime,
    DigitalArt,
    OilPainting,
    Watercolor,
    Sketch
}

pub trait PromptEngineer {
    fn enhance(prompt: Prompt) -> Prompt
    fn combine(prompts: List<Prompt>) -> Prompt
    fn to_provider_format(prompt: Prompt, provider: String) -> String

    docs {
        Utilities for prompt engineering.
        Transform and optimize prompts for different providers.
    }
}
```
</CodeBlock>

```toml
[spirit]
```

### Key Files

<CodeBlock title="src/conversation.dol" language="dol">
```dol
module chatbot/conversation @ 0.1.0

uses std/collections

pub enum MessageRole {
    System,
    User,
    Assistant,
    Tool
}

pub gen Message {
    has role: MessageRole
    has content: String
    has name: String?
    has tool_calls: List<ToolCall>?
    has tool_call_id: String?

    docs {
        A single message in a conversation.
        Supports all standard chat completion formats.
    }
}

pub gen Conversation {
    has id: UUID
    has messages: List<Message>
    has model: String
    has temperature: Float = 0.7
    has max_tokens: Int?

    is stateful
    derives Serializable

    fn add_message(role: MessageRole, content: String) -> Self
    fn get_context_window() -> List<Message>
    fn token_count() -> Int

    docs {
        A complete conversation with history management.
        Handles context window limits and message truncation.
    }
}
```
</CodeBlock>

<CodeBlock title="src/tools.dol" language="dol">
```dol
module chatbot/tools @ 0.1.0

pub gen ToolDefinition {
    has name: String
    has description: String
    has parameters: JsonSchema

    docs {
        Defines a tool that the LLM can invoke.
        Parameters follow JSON Schema format.
    }
}

pub gen ToolCall {
    has id: String
    has name: String
    has arguments: Json

    docs {
        An invocation of a tool by the model.
    }
}

pub trait ToolExecutor {
    fn execute(call: ToolCall) -> ToolResult
    fn validate(call: ToolCall) -> ValidationResult

    docs {
        Executes tool calls from the model.
        Implement this trait for each tool.
    }
}
```
</CodeBlock>

## Creating Custom Templates

You can create and share your own templates.

### Template Location

Custom templates are stored in:
```
~/.vudo/templates/<template-name>/
```

### Template Structure

<FileTree>
```
~/.vudo/templates/my-template/
+-- template.toml          # Template metadata
+-- files/                  # Files to copy
|   +-- project.dol
|   +-- src/
|   +-- ...
+-- hooks/                  # Optional scripts
    +-- post-create.sh
```
</FileTree>

<CodeBlock title="template.toml" language="toml">
```toml
[template]
name = "my-template"
description = "My custom project template"
version = "1.0.0"
author = "Your Name"

[variables]
# Variables that users can customize
project_name = { prompt = "Project name", required = true }
author = { prompt = "Author name", default = "Anonymous" }
license = { prompt = "License", default = "MIT", options = ["MIT", "Apache-2.0", "GPL-3.0"] }

[files]
# File transformations
"project.dol" = { template = true }  # Process as template
"README.md" = { template = true }
"src/*" = { copy = true }            # Copy as-is

[hooks]
post_create = "hooks/post-create.sh"
```
</CodeBlock>

### Using Template Variables

In template files, use `{{variable_name}}` for substitution:

<CodeBlock title="files/project.dol" language="dol">
```dol
module {{project_name}} @ 0.1.0

// Created by: {{author}}
// License: {{license}}

system {{project_name | pascal_case}} {
    docs {
        Main system for {{project_name}}.
    }
}
```
</CodeBlock>

### Installing Custom Templates

<Terminal title="Managing custom templates">
```
$ # Install from local directory
$ vudo template install ./my-template

$ # Install from git repository
$ vudo template install https://github.com/user/dol-template

$ # List installed templates
$ vudo template list
Built-in templates:
  basic       Minimal DOL project structure
  ai-image    AI image generation project
  ai-text     AI text/LLM project

Custom templates:
  my-template My custom project template (v1.0.0)

$ # Remove a template
$ vudo template remove my-template
```
</Terminal>

## Template Selection Tips

<Tip type="info">
**Choose the right template:**
- **basic** - General purpose, learning, or when you want minimal scaffolding
- **ai-image** - Image generation, art, visual AI projects
- **ai-text** - Chatbots, content generation, LLM applications
</Tip>

<Tip type="success">
**Template composition:** You can use `vudo new` with `--template` multiple times, or manually copy spirit files from one template to another.
</Tip>

<Tip type="warning">
**API keys:** The AI templates require API keys. Never commit `.env` files to git. The templates include proper `.gitignore` patterns.
</Tip>

## Common Issues

### Template Not Found

<Terminal title="Template error">
```
$ vudo new project --template enterprise
Error: Template 'enterprise' not found.

Available templates:
  basic, ai-image, ai-text

Install custom templates with:
  vudo template install <path-or-url>
```
</Terminal>

### Variable Substitution Errors

If template variables are not replaced:

```bash
# Check template.toml syntax
vudo template validate my-template

# Reinstall the template
vudo template remove my-template
vudo template install ./my-template
```

### Permission Errors

```bash
# Fix template directory permissions
chmod -R u+rw ~/.vudo/templates/
```

## Quick Reference

```
Template Commands
=================

CREATE PROJECT
  vudo new <name>                    # Use basic template
  vudo new <name> -t basic           # Explicit basic
  vudo new <name> -t ai-image        # AI image project
  vudo new <name> -t ai-text         # AI text project

  vudo template list                 # Show all templates
  vudo template install <path>       # Install template
  vudo template install <git-url>    # From repository
  vudo template remove <name>        # Remove template
  vudo template validate <name>      # Check template
```
