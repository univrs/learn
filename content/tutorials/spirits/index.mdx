# Spirit Development Track

Build secure, composable programs that run in the VUDO runtime. Spirits are WebAssembly modules with manifest metadata, WASM bytecode, and Ed25519 signatures that provide controlled access to system resources through a capability-based security model.

## What You'll Learn

By the end of this track, you will be able to:

- Understand the two-crate VUDO architecture (vudo_vm + spirit_runtime)
- Create and configure Spirit manifests in TOML format
- Work with all 14 capability types in the security system
- Use the 6 runtime modules for package management

## The VUDO Platform

The VUDO platform consists of two primary crates working together:

```
+------------------------------------------------------------------+
|                         VUDO Platform                             |
|                                                                   |
|  +---------------------------+   +---------------------------+    |
|  |        vudo_vm            |   |     spirit_runtime        |    |
|  |                           |   |                           |    |
|  |  Sandbox Runtime          |   |  Package Management       |    |
|  |  +---------------------+  |   |  +---------------------+  |    |
|  |  | WebAssembly Engine  |  |   |  | Manifest Parsing    |  |    |
|  |  | Fuel Metering       |  |   |  | Ed25519 Signatures  |  |    |
|  |  | Capability Checks   |  |   |  | Registry Storage    |  |    |
|  |  | Resource Limits     |  |   |  | Version Resolution  |  |    |
|  |  +---------------------+  |   |  | Dependency Graph    |  |    |
|  |                           |   |  | Credit Pricing      |  |    |
|  +---------------------------+   |  +---------------------+  |    |
|                                  +---------------------------+    |
|                                                                   |
|  Spirit Package = Manifest + WASM Bytecode + Ed25519 Signature    |
+------------------------------------------------------------------+
```

### Key Principles

1. **Two-Crate Design**: Separation of concerns between execution (vudo_vm) and management (spirit_runtime)
2. **Defense in Depth**: Multiple security layers with fuel metering and capability checks
3. **Cryptographic Identity**: Ed25519 signatures ensure package authenticity
4. **Credit Economics**: Resource usage tracked through the pricing module
5. **Minimal Capability**: Spirits only request capabilities they actually need

## Prerequisites Check

Before starting this track, verify your environment:

```bash
# Check VUDO installation
vudo --version

# Verify Rust toolchain
rustup show

# Ensure wasm32 target is installed
rustup target add wasm32-unknown-unknown

# Check cargo-component (for Spirit builds)
cargo component --version
```

If any commands fail, visit the [installation guide](/dol/install) to set up your environment.
