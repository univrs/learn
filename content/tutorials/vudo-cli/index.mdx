# VUDO CLI Track

Welcome to the VUDO CLI track. This track teaches you to master the full Spirit development lifecycle using the VUDO command-line interface.

## What is VUDO?

VUDO (Visionary Unified Development Operator) is the official CLI for Spirit development. It provides a unified interface for the complete development lifecycle:

- **Project scaffolding** - Create new Spirit projects from templates
- **Build and execution** - Compile to WASM and run in sandboxed environments
- **Cryptographic signing** - Ed25519 keypair generation and manifest signing
- **Registry operations** - Install, search, publish, and summon Spirits

```bash
# Quick install
cargo install vudo

# Verify installation
vudo --version
# vudo 0.2.3
```

## Quick Reference

| Command | Purpose |
|---------|---------|
| `vudo new` | Create new Spirit project from template |
| `vudo build` | Compile Spirit to WASM |
| `vudo run` | Execute Spirit in sandbox |
| `vudo test` | Run Spirit tests |
| `vudo sign` | Generate keypairs and sign manifests |
| `vudo pack` | Package Spirit for distribution |
| `vudo publish` | Publish Spirit to registry |
| `vudo install` | Install Spirit from registry |
| `vudo search` | Search the registry |
| `vudo info` | Get Spirit information |
| `vudo summon` | Run an installed Spirit |

## Registry Structure

VUDO maintains a local registry at `~/.vudo/`:

```
~/.vudo/
└── registry/
    ├── index.json
    ├── spirits/
    │   └── my-spirit/
    │       ├── 1.0.0/
    │       │   ├── manifest.json
    │       │   └── spirit.wasm
    │       └── latest -> 1.0.0/
    └── cache/
```
