# DOL Syntax Track

Welcome to the DOL Syntax track. This track provides a comprehensive introduction to the Design Ontology Language v0.8.0 syntax, focusing on its declarative nature and specification-first approach.

## What is DOL?

DOL (Design Ontology Language) is a **declarative specification language** for defining system ontologies. Unlike imperative programming languages that describe *how* to do things, DOL describes *what* things are and *how they relate* to each other.

```dol
// DOL is declarative: describe WHAT, not HOW
mod example @ 0.8.0

gen User {
    has id: u64
    has name: string
    has email: string
    has created_at: Timestamp
}
```

## Core Principles

DOL follows three fundamental principles:

1. **Specification First** - Define what should exist before implementing it
2. **Declarative Over Imperative** - Describe structure and relationships, not procedures
3. **Self-Documenting** - Every specification includes its own documentation

## DOL v0.8.0 Syntax Overview

This track covers the complete v0.8.0 syntax:

| Keyword | Purpose | Example |
|---------|---------|---------|
| `mod` | Module declaration | `mod container.runtime @ 0.8.0` |
| `gen` | Genome/type definition | `gen User { ... }` |
| `trait` | Behavior contract | `trait Lifecycle { ... }` |
| `system` | System architecture | `system OrderProcessing { ... }` |
| `rule` | Invariant constraints | `rule ValidOrder { ... }` |
| `evo` | Version migration | `evo V1 > V2 @ 2.0.0 { ... }` |
| `docs` | Documentation block | `docs { ... }` |
| `val` | Immutable binding | `val MAX_SIZE: u64 = 1000` |
| `var` | Mutable binding | `var counter: u64 = 0` |
| `forall` | Universal quantifier | `forall item in items { ... }` |
| `extends` | Trait extension | `trait Admin extends User { ... }` |

## Type System (Rust-Aligned)

| Type | Description |
|------|-------------|
| `string` | Text data |
| `bool` | Boolean values |
| `i8`, `i16`, `i32`, `i64` | Signed integers |
| `u8`, `u16`, `u32`, `u64` | Unsigned integers |
| `f32`, `f64` | Floating point |
| `Vec<T>` | Ordered collection |
| `Map<K, V>` | Key-value mapping |
| `Option<T>` | Optional values |
| `()` | Unit type (void) |
