In this chapter, you will initialize a new Spirit project using the `vudo new`
command with the `ai-image` template, which provides the foundation for AI-powered
content generation.

## Creating the Project

Open your terminal and run the following command:

<div className="rounded-xl overflow-hidden my-6" style={{ backgroundColor: 'var(--bg-secondary)', border: '1px solid var(--border-subtle)' }}>
  <div className="px-4 py-2 flex items-center gap-2" style={{ borderBottom: '1px solid var(--border-subtle)' }}>
    <Terminal className="w-4 h-4" style={{ color: 'var(--soft-gray)' }} />
    <span className="text-xs font-mono" style={{ color: 'var(--soft-gray)' }}>Terminal</span>
  </div>
  <pre className="p-6 text-sm font-mono overflow-x-auto" style={{ color: 'var(--glow-cyan)' }}>
{`vudo new lion-swan-creator --template ai-image`}
  </pre>
</div>

This command creates a new Spirit project with:
- AI image generation capabilities pre-configured
- Network access for API calls
- File storage for generated content
- Credit management hooks

## Project Structure

## Configure manifest.json

Navigate to your project and open `manifest.json`. Update it with the following
configuration:

<div className="rounded-xl overflow-hidden my-6" style={{ backgroundColor: 'var(--bg-secondary)', border: '1px solid var(--border-subtle)' }}>
  <div className="px-4 py-2 flex items-center gap-2" style={{ borderBottom: '1px solid var(--border-subtle)' }}>
    <FileCode className="w-4 h-4" style={{ color: 'var(--soft-gray)' }} />
    <span className="text-xs font-mono" style={{ color: 'var(--soft-gray)' }}>manifest.json</span>
  </div>
  <pre className="p-6 text-sm font-mono overflow-x-auto" style={{ color: 'var(--glow-cyan)' }}>
{`{
  "name": "lion-swan-creator",
  "version": "1.0.0",
  "description": "A Spirit that generates mythic Lion & Swan content",
  "author": "Your Name <you@example.com>",

  "spirit": {
    "type": "generator",
    "category": "art",
    "tags": ["mythology", "ai-art", "narrative"]
  },

  "capabilities": {
    "network": {
      "allowed_hosts": ["api.planetserve.io"],
      "max_requests_per_minute": 10
    },
    "storage": {
      "max_size_mb": 100,
      "allowed_types": ["image/png", "image/jpeg", "text/plain", "text/html"]
    },
    "credits": {
      "required": true,
      "min_balance": 5,
      "operations": {
        "generate_image": 3,
        "generate_text": 1
      }
    }
  },

  "ontology": {
    "entry": "src/ontology/main.dol",
    "validate_on_build": true
  },

  "commands": {
    "generate": {
      "handler": "src/handlers/generate.ts",
      "description": "Generate a new mythic scene"
    },
    "gallery": {
      "handler": "src/views/gallery.html",
      "description": "View generated content"
    }
  }
}`}
  </pre>
</div>

## Understanding the Configuration

Let us break down the key sections:

<div className="space-y-4 my-6">
  <div className="card">
    <h4 className="font-normal mb-2" style={{ color: 'var(--glow-cyan)' }}>spirit</h4>
    <p className="text-sm" style={{ color: 'var(--text-secondary)' }}>
      Defines the type and category of your Spirit. The <code>generator</code> type
      indicates this Spirit creates new content, and <code>art</code> category places
      it in the creative tools section.
    </p>
  </div>

  <div className="card">
    <h4 className="font-normal mb-2" style={{ color: 'var(--glow-cyan)' }}>capabilities.network</h4>
    <p className="text-sm" style={{ color: 'var(--text-secondary)' }}>
      Restricts network access to only <code>api.planetserve.io</code>, limiting potential
      security risks. The rate limit prevents accidental credit exhaustion.
    </p>
  </div>

  <div className="card">
    <h4 className="font-normal mb-2" style={{ color: 'var(--glow-cyan)' }}>capabilities.credits</h4>
    <p className="text-sm" style={{ color: 'var(--text-secondary)' }}>
      Specifies credit costs: 3 credits for image generation, 1 credit for text.
      The Spirit will check for <code>min_balance</code> before starting operations.
    </p>
  </div>

  <div className="card">
    <h4 className="font-normal mb-2" style={{ color: 'var(--glow-cyan)' }}>ontology</h4>
    <p className="text-sm" style={{ color: 'var(--text-secondary)' }}>
      Points to your main DOL file. When <code>validate_on_build</code> is true,
      the ontology is validated during the build process.
    </p>
  </div>
</div>
